# RT-026 Implementation: Claude Events to Kanban Bridge

## Summary
Designed and implemented the architecture for connecting Claude tool usage events to automatic Kanban board updates. When Claude creates/updates todos via `TodoWrite`, the Kanban board reflects these changes in real-time.

- Bridge architecture overview
- Event flow from ClaudeOutputParser to KanbanBoard
- Duplicate detection via content hashing
- Manual change preservation strategy
- Sub-agent task integration

### 2. `RickTerminal/Kanban/CardSource.swift`
New enum for tracking card origin:
```swift
enum CardSource: Codable, Equatable, Hashable {
    case manual                           // User created via UI
    case claude(sourceHash: String)       // Created from TodoWrite
    case ticket(ref: String)              // Imported from ticket system
    case subAgent(agentId: UUID, taskHash: String)
}
```

### 3. `RickTerminal/Claude/TodoWriteParser.swift`
Parser for Claude's TodoWrite output:
- `TodoItem` struct with content hash generation
- `TodoWritePayload` for batch todo processing
- `TodoWriteParser` with JSON, YAML, and line-format parsing
- Imperative to gerund conversion (`"Fix bug"` -> `"Fixing bug"`)

### 4. `RickTerminal/Claude/KanbanEventBridge.swift`
Main orchestrator component:
- Subscribes to `ClaudeOutputParser` events
- Creates/updates `KanbanCard` from `TodoItem`
- Handles sub-agent task tracking
- Preserves manual user changes
- Tracks duplicate via content hashing

## Architecture

```
ClaudeOutputParser
        │
        │ eventPublisher (Combine)
        ▼
KanbanEventBridge
        │
        ├── Parse TodoWrite → TodoWritePayload
        ├── Compute content hash for duplicate detection
        ├── Check knownHashes set
        │       │
        │       ├── [New] Create KanbanCard, add to board
        │       └── [Existing] Update if not manual
        │
        └── Handle sub-agent Task events
                │
                ├── [Started] Create card in "In Progress"
                └── [Completed] Move card to "Done"
        │
        ▼
KanbanBoard (@Published columns)
        │
        ▼
SwiftUI Views (automatic update)
```

## Key Design Decisions

### 1. Content-Based Hashing
- Uses SHA256 hash of normalized content for duplicate detection
- Status changes don't create new cards (same hash)
- Hash stored in `CardSource.claude(sourceHash:)`

### 2. Manual Change Preservation
- Cards with `source: .manual` never auto-updated
- `manuallyMovedCards` set tracks user-moved cards
- User can "claim" auto-generated cards

### 3. Real-Time via Combine
- Bridge subscribes to parser's `eventPublisher`
- Events processed on main thread
- Board's `@Published columns` triggers SwiftUI updates

## Integration Example

```swift
class SessionCoordinator {
    let parser: ClaudeOutputParser
    let board: KanbanBoard
    let bridge: KanbanEventBridge
    let agentManager: AgentColumnsManager

    init(session: ShellSession) {
        self.parser = ClaudeOutputParser(sessionId: session.id)
        self.board = KanbanBoard.standard(projectRef: session.workingDirectory)
        self.bridge = KanbanEventBridge(board: board)
        self.agentManager = AgentColumnsManager()

        // Connect components
        bridge.subscribe(to: parser)
        bridge.observeBoardChanges()  // Track manual moves
        agentManager.subscribe(to: parser)
    }
}
```

| TodoWrite events create/update cards | ✅ | `KanbanEventBridge.handleTodoWrite()` |

| Board updates in real-time | ✅ | Combine publishers + `@Published` |
| No duplicate cards created | ✅ | Content hash in `knownHashes` set |
